# 전문가 피드백 반영 내역

## 구조/배포

- `streamlit_app.py`가 `from app import main; main()`을 명시적으로 호출하도록 수정
- 로컬 실행, README, QUICKSTART, DEPLOY 문서를 모두 `streamlit_app.py` 기준으로 통일
- GitHub Actions 테스트 강화: 엔트리포인트, 공식 추천, JSON 검증, 퀴즈 커버리지, LaTeX 오류 패턴 검사

## 콘텐츠/유지보수

- 개념, 공식 규칙, FBD 시나리오, 문제은행, 학습 경로를 `content/*.json`으로 분리
- 기존 코드 내 하드코딩 중심 구조를 데이터 주도 구조로 개선
- 공식 데이터의 LaTeX 오류 일괄 수정
  - `\sump_1=\sump_2` → `\sum \mathbf p_1=\sum \mathbf p_2`
  - `F_{avg}\Deltat` → `F_{avg}\Delta t`
  - 극좌표 속도/가속도 표기 수정
  - `\sumF`, unicode escape 잔여 문자열 제거

## 공식네비게이터

- `st.code()` 표시를 `st.latex()` 렌더링으로 변경
- 단순 키워드 매칭에서 벗어나 부정 조건 처리 추가
  - `frictionless`, `no friction`, `마찰 없음` → 마찰 공식 오추천 억제
  - `not circular`, `원운동 아님` → 원운동 공식 오추천 억제
- 선택지와 규칙 연결 보강
  - 변위, 속력, 관성모멘트, 반력, 미지수 규칙 추가
- 추천 공식뿐 아니라 “이 공식이 아닐 수 있는 경우”와 “부정 조건 때문에 제외된 후보” 표시

## 설명 채점

- 기존 단순 키워드/길이 점수에서 개선
- 핵심 단어, 사용 조건, 공식 기호, 문제 힌트 반영
- 대표 오개념/위험 문장 탐지 추가
  - 속력이 일정하면 가속도 0
  - 수직항력은 항상 mg
  - 마찰은 항상 운동 반대
  - 구심력은 별도의 새로운 힘
  - 운동량 보존이면 운동에너지도 항상 보존

## 퀴즈/채점

- 36개 개념 각각에 최소 3개 이상 퀴즈 자동 연결
  - 개념 확인
  - 오개념
  - 적용
  - 공식 의미
- 퀴즈 필터 변경 시 이전 문제 상태 초기화
- 공식 감각 질문 채점 개선
  - 숫자 파싱
  - 분수 파싱
  - 증가/감소 방향 판정
  - “4배가 아니다” 같은 부정 표현 오답 처리

## 학습 기록/복습

- SQLite 기반 자동 저장 추가
- 개념별 숙련도 0~100 관리
- 정답/오답/설명 점수에 따라 숙련도 자동 변화
- 틀린 개념은 1일 후, 맞힌 개념은 숙련도에 따라 3/7/14일 후 복습 예약
- JSON 내보내기/가져오기 지원
- 가져오기 시 존재하지 않는 concept_id 제거 및 경고
- 주간/최근 기록, 약점 TOP, 복습 예정 표시

## AI/개인정보

- AI 피드백에 개념 제목/요약뿐 아니라 공식, 기호 뜻, 사용 조건, 흔한 실수, 문제 힌트, 예제까지 전달
- AI 문제 진단이 내부 공식네비게이터 결과를 함께 참고하도록 수정
- API 사용 시 어떤 데이터가 외부 API로 전송되는지 앱 안에 명확히 안내
- API 키가 없어도 핵심 학습 엔진이 작동하도록 오프라인 엔진 강화
