# DynaMap v3 UI/UX 수정 내역

전문가 UI/UX 피드백을 기준으로 앱 구조를 기능 나열형에서 학습 흐름형으로 재구성했습니다.

## 1. 메뉴 단순화

기존의 많은 독립 메뉴를 다음 6개로 묶었습니다.

- 홈
- 개념
- 문제
- 복습
- 리포트
- 설정

기존 기능은 제거하지 않고 아래처럼 재배치했습니다.

| 새 메뉴 | 포함 기능 |
|---|---|
| 홈 | 오늘의 학습, 빠른 시작, 최근 기록 |
| 개념 | 개념카드, 개념맵, 학습 경로, 시각화 |
| 문제 | 공식네비게이터, FBD 트레이너, 단계형 풀이 |
| 복습 | 퀴즈, 내 말로 설명, 헷갈림 목록 |
| 리포트 | 숙련도, 약점, 최근 기록, 다음 행동 제안 |
| 설정 | 백업, API 설정, AI 확장 도구 |

## 2. 홈 화면 개편

첫 화면을 통계 중심이 아니라 “오늘 무엇을 공부할지” 중심으로 바꿨습니다.

- 오늘의 목표
- 오늘 할 일 4단계
- 예상 소요 시간
- 오늘의 핵심 개념
- 최근 기록
- 빠른 시작 버튼

## 3. 문제풀이 단계형 UI

시험형 풀이를 한 화면 입력식에서 단계 진행식으로 바꿨습니다.

```text
문제 이해 → 물체 선택 → FBD → 좌표축 → 방정식 → 계산 → 해석
```

각 단계마다 현재 단계만 입력하고 점검할 수 있으며, 진행바를 추가했습니다.

## 4. 개념카드 구조 개선

개념카드는 “가볍게 시작하고 깊게 보기” 구조로 바꿨습니다.

바로 보이는 내용:

- 핵심 요약
- 대표 공식
- 언제 쓰는지
- 문제 힌트
- 바로 행동 버튼

접어서 보는 내용:

- 기호 설명
- 쉬운 비유
- 자주 하는 실수
- 연결 개념
- 미니 예제
- 개념별 시각화

## 5. FBD 트레이너 강조

FBD 트레이너를 문제 메뉴의 핵심 기능으로 재배치하고, 다음 피드백을 강화했습니다.

- 빠진 힘
- 불필요한 힘
- 좌표축 부족 요소
- 대표 방정식
- “구심력은 별도 힘이 아니다” 안내

## 6. 시각화 통합

시각화 랩을 별도 기능으로만 두지 않고, 개념카드와 개념 메뉴 안으로 연결했습니다.

예시:

- 원운동/곡선운동 → 원운동 벡터 시각화
- 마찰/수직항력/경사면 → 경사면 FBD 시각화
- 충돌/충격량/운동량 → 충돌 시각화
- 구름운동 → 구름운동 시각화
- 에너지/일/스프링 → 에너지 항 비교

## 7. 디자인 시스템 정리

다음 UI 요소의 스타일을 통일했습니다.

- 홈 히어로 영역
- 카드
- 태그/칩
- 공식 박스
- 단계 카드
- 오늘 할 일 카드
- 미니 지표 카드
- 모바일 반응형 레이아웃

## 8. 테스트 추가

기존 테스트에 더해 v3 UI 구조 확인 테스트를 추가했습니다.

- 6개 메인 메뉴 존재 여부
- 홈/문제 허브 연결 여부
- 단계형 풀이 UI 키워드 존재 여부

검증 명령:

```bash
python -m py_compile streamlit_app.py app.py dynacore/*.py
python -m pytest -q
```
